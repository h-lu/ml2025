小组综合项目：机器学习技术融合应用

项目背景

作为课程的收官项目，小组综合项目旨在让学生团队将本学期所学的多种机器学习技术（数据预处理、分类、回归、聚类、时间序列、模型可解释性等）融会贯通，解决一个真实或模拟的开放式商业问题。

项目目标

  1. 融合应用多种机器学习技术
  2. 完整经历机器学习项目全流程
  3. 将技术成果转化为实际商业价值
  4. 锻炼团队协作能力

建议选题方向

1. 用户行为分析与预测

  • 结合用户分群（聚类）与流失预测（分类+不平衡处理）
  • 应用 XAI 技术解释预测结果

2. 销售预测与优化

  • 结合销量预测（时间序列）与影响因素分析（回归+XAI）
  • 细分市场分析（聚类）与个性化预测

3. 风险控制与异常检测

  • 交易异常检测（无监督/监督学习）
  • 风险评分模型（分类/回归）

4. 产品推荐与个性化营销

  • 用户画像构建（聚类）
  • 购买倾向预测（分类）
  • 个性化推荐策略

项目开发流程

第一阶段：项目启动（第14周）

  1. 问题定义
    • 确定业务场景
    • 明确项目目标
    • 设计解决方案
  2. 数据准备
    • 数据收集/获取
    • 数据理解与探索
    • 初步预处理

第二阶段：模型开发（第15周）

  1. 特征工程
    • 特征提取与选择
    • 特征变换与组合
    • 数据清洗与标准化
  2. 模型构建
    • 实现多个机器学习模型
    • 模型集成与融合
    • 参数优化与调试
  3. 中期检查
    • 项目进度汇报
    • 技术难点讨论
    • 方向调整建议

第三阶段：完善与展示（第16周）

  1. 模型评估与优化
    • 全面的性能评估
    • 模型可解释性分析
    • 最终优化调整
  2. 结果展示与答辩
    • 项目展示准备
    • 成果汇报
    • 答辩问题准备

提交要求

1. 项目代码

  • 完整的 Jupyter Notebook/Quarto 文档（.ipynb/.qmd）
  • 包含所有数据处理、建模代码
  • 详细的注释和说明
  • 可重现的运行结果

2. 项目报告

  1. 项目背景与目标
    • 业务场景描述
    • 问题定义
    • 解决方案设计
  2. 技术实现
    • 数据处理流程
    • 特征工程方法
    • 模型构建过程
    • 评估与优化结果
  3. 业务价值
    • 模型效果分析
    • 商业价值转化
    • 实施建议
    • 未来展望

3. 展示文稿

  • 项目背景与目标
  • 技术方案概述
  • 关键结果展示
  • 商业价值阐述
  • 团队分工说明

评分标准

  • 技术融合（30%）：多种ML技术的合理运用
  • 实现质量（30%）：代码实现和模型效果
  • 商业价值（20%）：解决方案的实用性
  • 展示效果（20%）：报告质量和答辩表现

时间节点

  • 第14周：项目启动和方案设计
  • 第15周：中期检查
  • 第16周第一次课：最终展示与答辩
  • 第16周第一次课前：提交所有材料

注意事项

  1. 重视技术的融合应用，不要仅使用单一算法
  2. 注重项目的实际商业价值
  3. 合理分工，充分发挥团队协作
  4. 积极利用AI辅助工具提高开发效率
  5. 重视模型的可解释性分析
